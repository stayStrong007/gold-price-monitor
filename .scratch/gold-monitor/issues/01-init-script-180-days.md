# 01 · 一次性初始化脚本：灌入最近 180 天数据

Status: ready-for-agent

## 切片目标（tracer bullet）

改造现有 `fetch_history.js`，把写死的近两年日期区间改成「今天向前推 180 天」，跑一次，让 `data/gold_prices.json` 真实落地。

这是第一条端到端细线：**从抓取 → 到磁盘上一个有数据的 JSON 文件**。它给后面所有切片提供「真实数据」做地基。

## 完成的样子（可亲眼验证）

- 运行脚本后，`data/gold_prices.json` 存在。
- 文件里有 `黄金9999` + 7 个品牌，每个 `{name, data:[{date, price}]}`。
- `黄金9999` 覆盖最近约 180 天；品牌价按接口返回（可能多于/少于 180 天，本切片先不裁剪——裁剪逻辑属切片 02 的纯函数）。

## 不在本切片

- 合并/去重/裁剪逻辑（→ 02）。
- 任何前端、任何测试（本切片是一次性数据引导，靠手动跑脚本 + 肉眼看 JSON 验证）。

## 备注

- 时间按北京时间 UTC+8 归一为 `YYYY-MM-DD`，沿用现有脚本做法。
- 「今天」的日期由脚本运行时取系统时间，向前推 180 天生成 9999 的日期区间。
