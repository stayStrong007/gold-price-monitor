# 02 · 数据合并纯函数 mergeGoldData（测试接缝 A，主战场）

Status: ready-for-agent

## 切片目标（tracer bullet）

抽出并 TDD 实现核心纯函数 `mergeGoldData(oldData, freshData) → newData`，承载全部按 date 合并 / 去重 / 升序 / 保留全部历史 / 缺天补齐逻辑，**不含任何网络 I/O，不改入参**。

这是整个项目最值钱的一片——你最怕的「更新把历史冲掉」都在这里被挡住。用 red-green-refactor 逐个行为推进，不一次写完所有断言。

## 完成的样子（可亲眼验证）

测试全绿，覆盖以下外部行为（每条先红后绿）：

1. 同名品牌：旧数据 + 新数据按 `date` 合并，结果升序排列。
2. **重复日期**：同一天出现两个价，以新数据为准（新值覆盖旧值）。
3. **保留全部历史（不裁剪）**：旧数据里很久以前的天，合并后依然在——数据只增不减。
4. **漏跑自愈**：旧数据缺了中间某几天、新数据带来这几天时，缺口被补齐。
5. 某品牌数据**不足**（数据源没给那么多）时按实保留、不报错、不强行补齐。
6. 全程不触网、不改入参（喂构造数据，断言输出数据）。

## 不在本切片

## 备注

- 这是 PRD「测试接缝 A」。测试只验证输入→输出的外部行为，重命名内部变量不应使其失败。
- `today` 作为参数传入（而非函数内部取系统时间），以便测试可构造确定性场景。
